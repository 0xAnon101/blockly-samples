This sample shows the minumum code needed to define a new renderer that overrides connection shapes from the base renderer:

  1. Define a renderer that extends the base renderer: CustomNotchRenderer
  2. Register the renderer by name: 'custom_notch'
  3. Define a constants provider that extends the base constants provider: CustomNotchProvider. It contains all of the constants needed to size a block during rendering.
  4. Call the superclass constructor to set default property values.
  5. Override specific properties:
    • NOTCH_WIDTH: the width of previous and next connection notches.
    • NOTCH_HEIGHT: the height of previous and next connection notches.
  6. Override makeConstants_ to tell the custom renderer to use the custom constants provider.
  7. Override the makeNotch function to return a rectangular notch shape for previous and next connections.
  8. Override the makePuzzleTab function to return a rectangular tab for input and output connections.

makeNotch and makePuzzleTab are both called by the constants provider during initialization.
