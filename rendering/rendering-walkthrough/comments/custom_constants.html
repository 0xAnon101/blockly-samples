This sample shows the minumum code needed to define a new renderer that overrides constants from the base renderer:

  1. Define a renderer that extends the base renderer: CustomConstantsRenderer
  2. Register the renderer by name: 'custom_constants'
  3. Define a constants provider that extends the base constants provider: CustomConstantsProvider. It contains all of the constants needed to size a block during rendering.
  4. Call the superclass constructor to set default property values.
  5. Override specific properties:
    • NOTCH_WIDTH: the width of previous and next connection notches.
    • NOTCH_HEIGHT: the height of previous and next connection notches.
    • CORNER_RADIUS: the radius of all rounded corners on the block.
    • TAB_HEIGHT: the height of the puzzle tab used for input and output connections.
  6. Override makeConstants_ to tell the custom renderer to use the custom constants provider.

For a more complete list of constants, look at the base constants.js file.
